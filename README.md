## x3dpy_containerfield

The purpose of this repository is to create a patch for the existing code for the x3d package, as deployed as PyPI; that will enable the x3d.py to write appropriate containerField attributes when generating an XML encoding of a Python-based X3D scene.

 
- The x3d package Subversion based repository is at [x3d Sourceforge](https://sourceforge.net/projects/x3d/)

- The work in this patch is inspired by and  indebted to [andreasplesch/x3d-python-mod](https://github.com/andreasplesch/x3d-python-mod)

### Strategy:
The release from this project will be a modified X3duomToX3dPythonPackage.xslt stylesheet based on  [X3duomToX3dPythonPackage.xslt on Sourceforge](https://svn.code.sf.net/p/x3d/code/www.web3d.org/x3d/stylesheets/X3duomToX3dPythonPackage.xslt) at revision 38473. The release will also have a patch file, as generated by diff, that will allow later revisions of the version-control stylesheet to be updated.
